What part or parts of the reading did you find surprising?

I was surprised that the attribute value in HTML5 doesn't have to be enclosed in quotation marks as long as the value doesn't have spaces. I would assume that all coding would be made consistant. I plan to keep on using the quotation marks for all attributes as the books suggests.

What part or parts of the reading are still puzzling to you?

The one part I found puzzling was Aptana's HTML5 validator. I used both W3C's validator and Aptana's validator as suggested by the book. While my web page had no errors or warnings on W3C's validator, I had two warnings on Aptana's validator. I noticed that the two warnings were actually not warnings at all but the information about the page. I understand that Aptana uses a different validator but I am confused as to why it would count the information as warnings.

What part or parts of the reading did you find useful enough to want to incorporate its concepts?

One of the parts I found useful are the code comments. Comments will be useful for marking when a certain element starts and ends. They will also be helpful for testing layouts of an element by commenting out various code.

Another part I found useful are the validation tools. The validation tools will be helpful in finding any errors such as a missing closing tag or missing quotation marks.

I found the outline feature in Aptana to be the most useful. I would find it very helpful when trying to navigate through larger web pages. I can easily click the section I want versus scrolling through the entire document, only to miss the section I was looking for.

What part or parts of the reading was totally new to you?

I was unaware that whitespace adds to the size of the file. In the past I've used many spaces as a way of keeping my content organized. However, now that I know it will increase my file size, I will limit the amount of whitespace used.

Another new concept was Aptana. I have used Dreamweaver for quite some time but my current version is out of date. I have been looking for a good software to replace it since Adobe is now doing subscriptions. While Aptana may not have as many features as Dreamweaver, it is free and has most of the features I use often.

What part or parts of the reading did you already know? Did the reading increase your knowledge on the topic?

Since I started to teach myself code back in 2008, I knew a lot about this chapter. I have become quite familiar with the code structure and the basic layout of a web page. However, I did not know the formal names for certain parts of the code such as the document tree.

What part or parts of the reading did you find interesting?

I found it interesting that an id attribute is used to uniquely identify one element while a class attribute can be used on multiple elements. A few times when viewing a site's source code, I saw the code use the id attribute the same way as the class attribute. The website did not use the id to identify one element but many elements.
